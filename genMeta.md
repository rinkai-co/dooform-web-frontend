# Problem Statement

## Problem Identify <a href="#problem-identify" id="problem-identify"></a>

สำหรับปัญหาที่เกิดขึ้นตลอดระยะเวลาในการแปลเอกสารราชการ พบปัญหาส่วนใหญ่มาจากการที่ไม่มีมาตรฐาน/รูปแบบการแปลที่ชัดเจน

เราได้พบปัญหาซ้ำซากและตลาดที่<mark style="background-color:orange;">**ไม่มีมาตรฐานที่ชัดเจนในการแปลเอกสารราชการไทยเป็นภาษาอังกฤษ ซึ่งเป็นหนึ่งในสาเหตุหลักที่ทำให้ลูกค้าหลายรายถูกปฏิเสธวีซ่า**</mark>

หนึ่งในปัญหาหลักคือ <mark style="color:orange;">**ความซับซ้อนของรูปแบบเอกสารราชการไทย**</mark> ซึ่งมีลักษณะเฉพาะ เช่น คำราชาศัพท์ ฟอร์มที่ต้องจัดตำแหน่งให้ตรง ช่องว่างที่ต้องแปลตามลำดับ และรูปแบบการระบุข้อมูลที่ใช้กันเฉพาะในราชการไทย

{% hint style="warning" %}
ในร่างแผนธุรกิจนั้น ได้มีการเขียน Keyword หนึ่งคือ AI + Human Review ซึ่งอาจจะยังไม่ตรงตามฟังก์ชั่นของแอปพลิเคชั่นปัจจุบัน จากการพูดคุยนั้นยังไม่เห็นถึงความชัดเจนว่าจะนำ AI เข้ามาช่วยแปลเอกสารโดยตรง
{% endhint %}

### Bullet Summary <a href="#undefined" id="undefined"></a>

* ในช่วงการแปลเอกสารไม่มีมาตรฐานกลางที่กำหนดให้ชัดเจนในการแปล
* ความซับซ้อนของรูปแบบเอกสาร หากแปลผิดมีโอกาสที่จะถูกปฏิเสธวีซ่า
* ไม่มีระบบที่ช่วยให้สามารถควบคุมคุณภาพหรือจัดเก็บเอกสารอย่างมีประสิทธิภาพ
* ไม่มีระบบติดตามสถานะการแปลแบบเรียลไทม์

### Problem Statement <a href="#problem-statement" id="problem-statement"></a>

เราสามารถตั้ง problem statement ได้จากหลาย persona เช่น user/stakeholder หากเป็นในส่วนของ user

> ในฐานะที่เป็นผู้แปลเอกสาร, การจัดการกับฟอร์แมตเอกสารเป็นสิ่งที่ทำให้เสียเวลา
>
> ในฐานะเจ้าของธุรกิจ, พนักงานแปลเอกสารมักจะใช้เวลาจัดการกับหน้ากระดาษมากกว่าการแปล

# Solutions

โปรดดูเอกสาร [problem-statement.md](problem-statement.md "mention")

## Solution <a href="#solution" id="solution"></a>

สร้างระบบที่อนุญาตให้ผู้ใช้งานกรอกข้อมูลเพียงอย่างเดียว ไม่ต้องจัดการกับปัญหาฟอร์แมตเอกสาร โดยลิสต์ด้านล่างเป็นระบบที่สามารถเกิดขึ้นได้

* ระบบที่ให้ user กรอกฟอร์มแล้ว generate complete pdf/word
* ตรวจสอบประวัติการกรอกฟอร์ม
* คลังข้อมูลกลาง
  * ประวัติงานแปลทั้งหมด
  * คลังความรู้ภายในองค์กร
* ระบบตรวจสอบงานแปล
* Ticket System ที่ให้ user สามารถสร้าง case และแชร์งานได้ผ่านลิงก์ชั่วคราว
  * สามารถตรวจสอบสถานะการแปลได้
* In-line Suggestion by AI
* AI Autofill

## Further Idea <a href="#further-idea" id="further-idea"></a>

ในอนาคตข้างหน้า หากระบบมีการ Implement CRM Service ที่มีการเก็บข้อมูลลูกค้า การดึง API จากการแปลก็อาจจะช่วยให้ผลิตเอกสารออกมาได้ไวขึ้นโดยที่ไม่ต้องกรอกข้อมูลซ้ำ ๆ

* API Adapter with CRM
  * หากเป็นบริษัทที่มีการออกเอกสารจำนวนมากให้กับพนักงาน/ลูกค้าต่อเดือน และบริษัทมีข้อมูลในมือแล้ว บริษัทสามารถนำ API ไปสร้างเอกสารแบบ automation ได้
* In-line Suggestion
  * แนะนำการแปลว่าบรรทัดนี้อาจเกิดข้อผิดพลาด เช่น การสะกดชื่อจังหวัด ที่อยู่ หรือ ไวยากรณ์ผิด
* OCR
  * เลือกประเภทเอกสารและอัปโหลด หลังจากนั้นระบบตรวจสอบข้อความที่ปรากฎในภาพและทำการพิมพ์ให้อัตโนมัติ

## Arguments

ในการ implement product ออกมาใหม่นั้น จำเป็นที่จะต้องคำนึงถึงช่องโหว่และจุดด้อยที่อาจเกิดขึ้นในการพัฒนาและต้องสามารถตอบได้ว่าทำเพื่ออะไร โดยหัวข้อในด้านล่างจะเป็น argument ที่อาจจะเกิดขึ้นได้ในระหว่างการ implement functions

### Current Functionality <a href="#current-product-functionality" id="current-product-functionality"></a>

เพื่อความเข้าใจที่ตรงกัน ตัวโปรดัก Dooform ปัจจุบันมีฟังก์ชั่นคือ <mark style="color:orange;">**ให้คนเข้ามากรอกข้อมูลลงในฟอร์มแล้ว export โดยไม่ต้องยุ่งกับ format**</mark>

ฟังก์ชั่นหลักของแอปพลิเคชั่น Dooform คือ <mark style="background-color:yellow;">User สามารถพิมพ์ข้อมูลภาษาอังกฤษเข้าเว็บไซต์แล้วได้รับไฟล์ Docx/PDF โดยที่ไม่ต้องใช้งานแอปพลิเคชั่น Word ทำให้หมดปัญหาเรื่อง Format ที่ไม่เป็นมาตรฐาน</mark>

<details>

<summary>ใช้แอปนี้กับ word จะต่างกันยังไง มันก็เหมือนกัน?</summary>

สำหรับข้อโต้แย้งนี้ เราอาจจะกลับมาดูที่ปัญหาของเราคือ format ใน docx file มีความเชื่อว่าผุ้ที่แปลเอกสารเสียเวลาในการจัดฟอร์แมตเอกสารค่อนข้างนาน และเอกสารที่ทำออกมาในแต่ละครั้งจะไม่มีมาตรฐานกลางที่เท่ากัน คคนแปลต้องมาเสียเวลาจัดขนาดฟอนต์ กรอบ รูปแบบอื่น ๆ ด้วยตนเอง

</details>

<details>

<summary>แก้แค่ปัญหาเรื่อง format จะคุ้มกับการลงทุนไหม?</summary>

ในส่วนนี้อาจจะต้องมีการพิจารณาพูดคุยเพื่อความชัดเจนอีกครั้งว่า ฟังก์ชั่นที่ให้คนไปกรอกข้อมูลเองโดยไม่ต้องยุ่งกับฟอร์แมตมันทำให้ชีวิตคนสะดวกสบายขึ้นในระดับที่เขาจะยอมเสียเงินเพื่อแลกกับปัญหาฟอร์แมตหรือไม่? หากเรานำ AI เข้ามาช่วยแปล จะเป็นการสร้าง value ให้กับโปรดักส์ และช่วยลดแรงงานมนุษย์ได้หรือไม่

</details>
